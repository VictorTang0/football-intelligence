# 顶级模型大版本进化与推演质量协议 (v4.0)

本协议适用于 Football Intelligence 项目的所有今日预测 (Command A)、盘口更新 (Command B) 及赛后进化 (Command C) 工作流。

---

## 一、 双核独立推演架构

1. **物理预测系统 (M01-M09)**：
   - 基于球队硬实力、伤停阵容、近期战效、战术匹配、主客场草皮与气候等客观基本面，通过双变量泊松分布计算真实期望进球数与基本面预测结论。

2. **M10 独立竞彩大师系统**：
   - **角色定位**：沉浸竞彩大盘多年的独立精算大师，具备自主思考与反向自我进化机制。
   - **四维覆盖**：推演全面覆盖 **【胜平负/方向】**、**【进球数】**、**【比分】**、**【半全场】** 四大维度。
   - **输出形态**：根据竞彩官方即时水位、水温轨迹、背离警报与大单资金倾向，独立输出正路热爆、机构避险或冷门爆冷结论。
   - **显示原则**：在对应项后标注 `(竞彩首选)`。**不强制要求每项都得出**，唯有当 M10 竞彩大师捕捉到明确信号、具备充足信心时方可显示。
   - **独立性**：M10 竞彩大师结论相对独立，【不强制受物理预测系统方向束缚】，允许展现与主预测方向不一致的机构避险或爆冷选项。

---

## 二、 2x4 顶栏微型表格与四维独立核销

1. **顶栏 2x4 微型表格**：
   - 位于【今日场次】后、【方向准确率】前，以 2 行 x 4 列微型表格分别实时展现 M10 在【方向】、【进球】、【比分】、【半全】的 `(竞彩首选)` 独立命中率。
2. **四维独立核销**：
   - 赛后核销时，系统自动清洗落盘，在 `history.json` 的 `sporttery_primary_stats` 结构中独立统计 M10 在四大维度的命中率与进化记录。

## 四、 4大高质量原则落地细则

1. **最高质量与分级推演**：
   - 推演理据建立“物理战术层 $\rightarrow$ M10 水温与机构博弈层 $\rightarrow$ 策略交付层”三层深剖结构。
   - 赛后进化建立 P1 盘口陷阱识别偏差 / P2 信息缺失 / P3 偶发红牌与极端事件三级归因。
2. **防主观臆断与数据解耦**：
   - 严格解耦 `[物理事实]` (官方战绩/即时水位) 与 `[模型推算]` (庄家倾向/期望估算)。
   - 数据缺失时显式标注 `[数据尚待补充]`，严禁凭空虚构；遇未完赛/无法查到赛果的比赛触发防抹除保护并向用户询问处置。
3. **Top 1% 精英视角与反向防坑**：
   - 深度剖析新手易犯的“名气/排名依赖症”与“盲目追热门大胜”误区；显式展现正反方优劣势对比。
4. **完成度自查与弱点诚实**：
   - 置信度较低 (Confidence < 55%) 赛事诚实指出模型弱点，并由 M10 竞彩大师给出【替代避险/冷门路线】。
   - 强一致性校验物理与博弈分析逻辑，绝不为凑一致性修改 M10 的独立盘口结论。

## 五、 数据真实性、交锋与近期战绩权威自检协议 (Zero-Hallucination Protocol)

1. **绝对禁令：彻底根除伪随机战绩生成**：
   - **严禁任何伪随机/Seed 数据拼装函数 (如 gen_local)**。所有交锋历史 (H2H) 与近期战绩 (Recent Matches) 必须 100% 来源于体彩官方开奖数据库 (`official_sporttery_2022_2026.json`) 或竞彩网 API 直抓。

2. **近期战绩 100% 真实性与严格倒序自检**：
   - 近期战绩必须且只能提取该球队**真实参加且已完场的官方比赛**。
   - 提取列表必须**严格按比赛日期 (date) 降序排列**，确保第 1 场为真正时间最近的比赛，依次列出最近 5 场。

3. **竞彩网排名/积分缺失时的标准应对 Protocol**：
   - **第一优先**：从全量 1.9 万场体彩官方开奖真理库中，按当赛季联赛独立实时聚合计算真实积分、排名与净胜球。
   - **第二优先**：若属于无官方开奖历史的新联赛或杯赛，在 `standing_note` 显式标注 `[竞彩网暂未公布即时积分，已标记为数据更新中]`。
   - **底线**：严禁凭空假想或凑数填写排名与胜率。
